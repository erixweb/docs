Personalitza el teu editor de codi per millorar la teva experiència de desenvolupament amb Astro i desbloquejar noves funcionalitats.

## VS Code

[VS Code](https://code.visualstudio.com/) és un editor de codi popular per a desenvolupadors web, desenvolupat per Microsoft. El motor de VS Code també s'utilitza en editors de codi al navegador com [GitHub Codespaces](https://github.com/features/codespaces) i [Gitpod](https://gitpod.io/).

Astro funciona en qualsevol editor de codi. No obstant això, recomanem utilitzar VS Code per als teus projectes d'Astro. Mantenim l'[extensió oficial d'Astro per a VS Code](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode), la qual desbloqueja noves funcionalitats i millora l'experiència de desenvolupament pels teus projectes.

- Resaltat de sintaxis per a fitxers `.astro`.
- Informació de tipus de TypeScript per a fitxers `.astro`.
- [Intellisense de VS Code](https://code.visualstudio.com/docs/editor/intellisense) per a autocompletat, suggeriments i més.

Per començar, instal·la l'[extensió d'Astro per a VS Code](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode).

📚 Apren com [configurar TypeScript](/ca/guides/typescript/) en el teu projecte de Astro.

### IDE de JetBrains

El suport inicial per a Astro va arribar al Webstorm 2023.1. Pots instal·lar el connector oficial a través de la [pàgina de connectors de Jetbrains](https://plugins.jetbrains.com/plugin/20959-astro) o cercant "Astro" a la pestanya de Connectors i beneficiar-te de les funcions com ressaltat de sintaxi, formateig i autocompletat de codi, amb plans per afegir més funcions avançades en el futur. També està disponible per a les altres [IDEs de Jetbrains amb suport per a Javascript](https://www.jetbrains.com/products/#lang=js&type=ide).

El proper [Fleet IDE](https://www.jetbrains.com/fleet/) de JetBrains també donarà suport als servidors de llenguatge i, com a consequencia, les nostres eines actuals podran executar-se allà sense cap problema.

## Altres editors de codi

La nostra increïble comunitat manté extensions per a altres editors de codi, incloent:

- [Extensió de VS Code per a Open VSX](https://open-vsx.org/extension/astro-build/astro-vscode)<span style="margin: 0.25em;"><Badge variant="accent">oficial</Badge></span> - L'extensió oficial de VS Code, està disponible al registre d'Open VSX per a altres plataformes com [VSCodium](https://vscodium.com/).
- [Extensió de Nova](https://extensions.panic.com/extensions/sciencefidelity/sciencefidelity.astro/)<span style="margin: 0.25em;"><Badge variant="neutral">Comunitat</Badge></span> - Proveeix ressaltat de sintaxi i autocompletat per a Astro a Nova.
- [Vim Plugin](https://github.com/wuelnerdotexe/vim-astro) <span style="margin: 0.25em;"><Badge variant="neutral">Comunitat</Badge></span> - Proveeix ressaltat de sintaxi, indentació i compatibilitat amb folding de codi per a Astro a Vim o Neovim.
- Neovim [LSP](https://github.com/neovim/nvim-lspconfig/blob/master/doc/server_configurations.md#astro) i [TreeSitter](https://github.com/virchau13/tree-sitter-astro) Plugins <span style="margin: 0.25em;"><Badge variant="neutral">Comunitat</Badge></span> - Proveeix ressaltat de sintaxi, anàlisi d'arbres i autocompletat per a Astro a Neovim.
- Emacs - Vegeu les instruccions per a [Configurar Emacs i Eglot](https://medium.com/@jrmjrm/configuring-emacs-and-eglot-to-work-with-astro-language-server-9408eb709ab0) <span style="margin: 0.25em;"><Badge variant="neutral">Comunitat</Badge></span> per a treballar amb Astro.
- [Ressaltat de sintaxi d'Astro per a Sublime Text](https://packagecontrol.io/packages/Astro) <span style="margin: 0.25em;"><Badge variant="neutral">Comunitat</Badge></span> - El paquet Astro per a Sublime Text, disponible al gestor de paquets de Sublime Text.

## Editors de codi del navegador

A més dels editors de codi locals, Astro també funciona als editors de codi al navegador, incloent:

- [StackBlitz](https://stackblitz.com/) i [CodeSandbox](https://codesandbox.io) - Editors de codi en línia al navegador, amb ressaltat de sintaxi incorporat per a fitxers `.astro`. No cal instal·lació ni configuració!
- [GitHub.dev](https://github.dev/) - Et permet instal·lar l'extensió d'Astro per a VS Code com a [extensió web](https://code.visualstudio.com/api/extension-guides/web-extensions), la qual t'ofereix accés només a algunes de les característiques de l'extensió completa. Actualment, només dóna suport al ressaltat de sintaxi.
- [Gitpod](https://gitpod.io/) - És un entorn de desenvolupament a la núvol on pots instal·lar l'extensió oficial de VS Code des de l'Open VSX.

## Altres eines

### ESLint

[ESLint](https://eslint.org/) és un linter popular per a JavaScript i JSX. Per activar la compatibilitat amb Astro, pots instal·lar [un plugin mantingut per la comunitat](https://github.com/ota-meshi/eslint-plugin-astro).

Consulta [la guia de l'usuari del projecte](https://ota-meshi.github.io/eslint-plugin-astro/user-guide/) per obtenir més informació sobre com instal·lar i configurar ESLint per al teu projecte.

### Stylelint

[Stylelint](https://stylelint.io/) és un linter popular per a CSS. Existeix una [configuració de Stylelint mantinguda per la comunitat](https://github.com/ota-meshi/stylelint-config-html) que ofereix suport per a Astro.

Les instruccions d'instal·lació, integració de l'editor i informació addicional es poden trobar al fitxer README del projecte.

### Prettier

[Prettier](https://prettier.io/) és un formatejador popular per a JavaScript, HTML, CSS i més. Si estàs utilitzant [l'extensió de VS Code d'Astro](https://marketplace.visualstudio.com/items?itemName=astro-build.astro-vscode) o [el servidor de llenguatge d'Astro dins un altre editor](#altres-editors-de-codi), el formateig de codi amb Prettier està inclòs.

Per donar suport al formateig de fitxers `.astro` fora de l'editor (per exemple, CLI) o dins d'editors que no donen suport a les nostres eines d'editor, instal·la [el plugin oficial de Prettier d'Astro](https://github.com/withastro/prettier-plugin-astro).

Per començar, primer instal·la Prettier i el plugin:

<PackageManagerTabs>
  <Fragment slot="npm">
  ```shell
  npm install --save-dev prettier prettier-plugin-astro
  ```
  </Fragment>
  <Fragment slot="pnpm">
  ```shell
  pnpm install -D prettier prettier-plugin-astro
  ```
  </Fragment>
  <Fragment slot="yarn">
  ```shell
  yarn add --dev prettier prettier-plugin-astro
  ```
  </Fragment>
</PackageManagerTabs>

Prettier detectarà automàticament el plugin i ho utilitzarà per processar els fitxers `.astro` quan ho executis:

```shell
prettier --write .
```

Consulta el [README del plugin de Prettier](https://github.com/withastro/prettier-plugin-astro/blob/main/README.md) per obtenir més informació sobre les seves opcions, com configurar Prettier dins de VS Code i més.

:::caution[Us amb pnpm]
Degut a problemes dins de Prettier, el plugin no es detectarà automàticament quan s'utilitza [pnpm](https://pnpm.io/). Perquè trobi el plugin, s'ha d'afegir el següent paràmetre en executar Prettier:

```shell
prettier --write . --plugin=prettier-plugin-astro
```
:::
